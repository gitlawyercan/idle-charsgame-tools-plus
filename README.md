# Idle-charsgame-tools-plus
适用于  https://idle.charsgame.com  游戏的一款油猴插件
# Project Idle 自动填表 + 数据统计（增强版）

一个功能强大的 Tampermonkey 油猴脚本，专为 [Project Idle](https://idle.charsgame.com) 游戏设计，集成账号登录管理、背包物品统计、技能等级统计、市场买单采集等功能。

---

## 📦 安装方法

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 浏览器扩展
2. 新建脚本，将本代码完整粘贴后保存
3. 打开游戏页面，脚本会自动加载

---

## 🚀 功能特性

### 1. 🔐 账号登录管理
- **自动填表登录**：一键填充账号密码并自动提交登录
- **账号分组管理**：内置 6 个分组（砍伐、种植、采矿、打猎、锻造、纺织），支持自定义分组
- **账号磁条拖动**：拖拽账号按钮移动到其他分组
- **账号/分组重命名**：双击名称可编辑修改
- **删除功能**：软删除账号或整个分组
- **记住上次登录账号**：高亮显示最后一次使用的账号
- **右键填充**：右键点击只填充表单不自动提交

### 2. 📦 背包物品统计
- **自动采集**：通过 API 获取当前登录账号的背包物品
- **全账号汇总**：统计所有已记录账号的背包物品总量
- **分组/单账号筛选**：按账号分组或单个账号过滤查看
- **搜索过滤**：按物品名称搜索
- **导出数据**：支持 JSON / CSV 格式导出

### 3. 🎯 技能等级统计
- **自动采集**：通过 API 获取技能等级 + **金币/钻石**
- **全账号对比表**：横向对比所有账号的各项技能等级（平均值/最高值）
- **单账号查看**：查看指定账号的技能详情
- **导出数据**：支持 JSON / CSV 格式导出

### 4. 🛒 市场买单采集
- **API 自动采集**：遍历 64 种材料/资源类物品的买单数据
- **按总价排序**：自动计算单价×数量，按总价降序排列
- **搜索过滤**：按物品名称/ID 搜索
- **定时采集**：可开启每 1 小时自动采集
- **导出数据**：支持 JSON / CSV 格式导出

### 5. 🎛️ 面板交互
- **悬浮面板**：固定在页面任意位置
- **可拖拽**：拖拽面板标题栏移动位置
- **折叠/展开**：点击箭头或双击标题栏
- **设置菜单**：导出/导入配置、重置位置、清除数据、退出登录
- **底部状态栏**：实时显示物品种类数、技能数、金币数

## 🔧 技术原理

### 账号登录管理
- **账号配置**：内置账号列表以 JS 对象硬编码在脚本中，自定义账号保存在 `localStorage`
- **填表登录**：通过 DOM 选择器查找邮箱/密码输入框，用 JS 原生 value setter 填充值，触发 `input`/`change` 事件，模拟点击登录按钮
- **软删除**：在 `localStorage` 中记录被删除的账号/分组，渲染时过滤
- **拖拽移动**：HTML5 原生 `dragstart` / `dragover` / `drop` 事件，将账号从一个分组移到另一个分组后重绘面板
- **重命名**：`localStorage` 保存 `邮箱→新名称` / `原分组名→新分组名` 的映射表

### 背包物品统计
- **数据获取**：调用 API 获取背包数据，携带 JWT Token 认证
- **ID→名称转换**：API 返回 `itemId`（数字），通过内置 419 个物品映射表转中文名称
- **缓存存储**：以邮箱为 key 存入 `localStorage`，包含物品清单和时间戳
- **聚合统计**：读取所有账号数据，按物品名称汇总数量，支持分组/账号筛选

### 技能等级统计
- **数据获取**：调用 API 获取技能等级 + 金币/钻石，携带 JWT Token
- **ID→名称转换**：API 返回 `actionSkillId`（数字），通过内置 27 个技能映射表转中文名称
- **对比表**：构建 `技能名→{各账号等级}` 映射，计算平均值和最大值，渲染横向对比表格
- **单账号视图**：筛选后按等级降序展示

### 市场买单采集
- **数据获取**：调用 API，请求体为 `{itemId: xxx}`，携带 JWT Token
- **遍历采集**：内置 64 个目标物品 ID，逐个请求，间隔 1.2 秒避免限流，跳过已采集物品
- **总价计算**：`单价 × 数量` 得到小计，汇总为该物品总价，按总价降序排列
- **定时采集**：`setInterval` 每 1 小时自动重新采集

### 面板交互
- **面板创建**：动态创建 `div` 元素并设置内联样式，插入 `document.body`
- **拖拽**：监听 `mousedown` / `mousemove` / `mouseup` 计算偏移量更新位置
- **折叠**：切换 `display: none/block`，状态存入 `localStorage`
- **位置恢复**：页面刷新时从 `localStorage` 读取位置和折叠状态

## 🌐 API 接口

### 统一认证方式

Token 来源：`localStorage` 中 `project-idle-auth.state.token`

请求头格式：`Authorization: Bearer <token>`

### 接口列表

| 接口 | 用途 | 请求方法 | 请求体 | 关键返回字段 |
|---|---|---|---|---|
| `/api/v1/player/profile` | 技能等级 + 金币/钻石 | POST | `{}` | `actionSkills[]`, `gold`, `diamonds` |
| `/api/v1/inventory` | 背包物品 | POST | `{}` | `inventory[].itemId`, `inventory[].qty` |
| `/api/v1/market/order-book` | 市场买单 | POST | `{ "itemId": xxx }` | `buyOrders[].price`, `buyOrders[].qty` |

### 备用方案

如果 API 调用失败（Token 失效/网络异常），脚本会自动回退到 DOM 解析方式（解析页面 HTML 元素获取数据），保证核心功能不中断。

## 📂 数据存储

所有数据存储在 `localStorage` 中，以下是存储的 key：

| Key | 用途 |
|---|---|
| `project_idle_last_account` | 上次登录账号 |
| `project_idle_custom_accounts` | 自定义账号配置 |
| `project_idle_inventory_data` | 背包物品数据 |
| `project_idle_skill_data` | 技能等级数据 |
| `project_idle_buy_orders_data` | 市场买单数据 |
| `project_idle_group_renames` | 分组名称修改记录 |
| `project_idle_account_renames` | 账号名称修改记录 |
| `project_idle_deleted_accounts` | 已删除账号记录 |
| `project_idle_deleted_groups` | 已删除分组记录 |
| `project_idle_panel_collapsed` | 面板折叠状态 |
| `project_idle_panel_left` | 面板位置 X |
| `project_idle_panel_top` | 面板位置 Y |
